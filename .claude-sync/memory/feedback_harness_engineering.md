사용자는 하네스 엔지니어링(Harness Engineering) 방식으로 프로젝트를 운영하길 원함.

**핵심 원칙:**
- "Human On the Loop" — 사용자는 방향(Why)을 정하고, 에이전트가 실행(How)을 자율적으로 수행
- 에이전트끼리 역할을 나눠 협업 (아키텍트, 코더, 테스터, 리뷰어 등)
- 결과물이 불만족스러우면 출력물이 아니라 하네스(프로세스/제약)를 개선
- 외부 URL, 검색, 커뮤니티, 공식 문서 자유롭게 참조 가능
- 외부 도구/라이브러리 자유롭게 다운로드 및 사용 가능

**Why:** Martin Fowler 블로그의 harness engineering / humans-and-agents 글 + 한국어 Claude Code 동작원리 글에서 영감받음. 사용자는 프로젝트가 에이전트 주도로 완벽하게 완성되길 원함.

**How to apply:**
- 매 작업마다 적절한 에이전트 역할 분담 (Plan → Code → Test → Review)
- 독립적 작업은 반드시 병렬 Agent 호출 (리서치, 코드분석, 외부서치 동시)
- 코드 수정 후 별도 검증 에이전트가 교차 검증 (자기 검증 X)
- CLAUDE.md를 하네스의 핵심 제약 문서로 유지/업데이트
- 품질 문제 발견 시 코드만 고치지 말고 프로세스/린터/테스트도 함께 개선
- 커밋 전 반드시 검증 단계 포함
- LLM 판단보다 린터, 테스트, 실제 실행 결과를 우선
- 2-3회 실패 시 사용자에게 에스컬레이션
