## v29 모델 품질 문제 (2026-03-20 진단)

v29 모델로 변환한 결과물에서 기계음, 마찰음, 음정불안정 문제 발생.

**Why:** 23개 소스 파일(3개 v28 + 20개 mp3record) 학습. MP3 소스 + 결함있는 증강 기법 조합.

## 근본 원인 (우선순위 순)

1. **asetrate 피치 시프트 증강** (치명적) — 제거됨
   - asetrate = 재생속도 변경 후 리샘플 → 피치 올라가지만 템포 12% 빠름
   - 모델이 "고음 = 빠른 발음"으로 잘못 학습 → 기계적 피치 전환

2. **SLICE_DURATION=3.5s** (심각) — 5.0s로 증가
   - 한국 가요 프레이즈 중간 절단 → 비브라토/멜리스마 미학습

3. **고음 오버샘플링 과다** (심각) — 축소 (2/1/0 → 1/0/0)
   - MP3 아티팩트 세그먼트 반복 학습 → 기계음 강화

4. **adeclick threshold=2** (중간) — threshold=5, burst=4로 변경
   - 비브라토 진폭변화/자음 포먼트도 클릭으로 오탐 → 마찰음 유발

5. **MP3 소스 HF 아티팩트** (중간) — 전처리에 15.5kHz lowpass 추가
   - MP3 인코더 HF 노이즈를 음색으로 학습 → 금속성 고음

6. **Crepe F0 + 화음곡** (중간) — UI에 경고 추가
   - 화음에서 피치 추적 혼란 → 음정 불안정
