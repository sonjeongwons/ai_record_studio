# 변환 가이드 — AI Voice Studio (app.py 클라이언트 기준)

> **이 문서는 `python app.py` 실행 후 나타나는 클라이언트 UI 기준으로 작성되었습니다.**
> 버전 기준: **v73** (2026-04-29)

## 1. 신규 전처리 / 모델 학습이 필요한가?

### 단기 (즉시 테스트 가능)

**신규 학습 불필요.** 현재 **My Voice v50** 모델에서 변환 파라미터(index_rate, protect)만 바꿔 재변환하면 됩니다.

v73에서 바뀐 것은 변환 파라미터이지 학습 데이터나 모델 자체가 아닙니다:
- `index_rate` 0.50 → **0.75** (AI스러움의 직접 원인, 커뮤니티 권장값)
- `protect` 0.40 → **0.33** (커뮤니티 남성 노래 표준; 자음 보호 강화)
- `fcpe.pt` Docker 이미지에 추가 → FCPE 방식이 MR만 나오던 버그 해결

### Docker 이미지 업데이트 필요

FCPE 수정은 Dockerfile이 변경되었으므로 GitHub Actions Docker 빌드가 완료된 후 RunPod 엔드포인트의 이미지를 새 태그로 업데이트해야 합니다.

> RunPod 대시보드 → 엔드포인트 설정 → Docker 이미지 태그 최신으로 변경 → 저장

### 여전히 AI스러움이 심하다면? (재학습 고려)

v57 모델 분석 결과 v50 모델은 과적합 징후가 발견되었습니다:
- `attn_layers.3.conv_k.bias` absmax = **6.61** (정상의 약 10배)
- FAISS 실효 차원 = 45/768 (협소 → AI스러움 유발)

이 경우 **같은 소스음원으로 재학습**하되, 조기 종료를 적용합니다:
- 타겟 에폭: **70~75** (loss_g 4.0 이상 구간에서 best_epoch 선택)
- 그 외 학습 파라미터는 변경 없음 (같은 소스음원, 같은 전처리)
- 기존 전처리 결과물을 재사용할 수 있어 전처리는 다시 실행할 필요 없음

## 2. 변환 탭 전체 파라미터 설명

| UI 항목 | 기본값 | 설명 |
|--------|-------|------|
| **피치 조절** | 0 | 반음 단위. 0 = 원곡 그대로. 삑사리 시 -2~-3 시도 |
| **인덱스 비율** | 0.75 | AI 모델 목소리 강도. 0.70~0.80 권장. 화음/듀엣은 0.20~0.35 |
| **피치 추출 방식(F0)** | RMVPE | 대부분 RMVPE. 팔세토/고음 많은 곡은 FCPE |
| **언어** | 자동 | 한국어 → `ko`, 영어 → `en`, 모르면 `자동` |
| **노이즈 제거** | OFF | 변환 후 기계음이 들릴 때만 ON |
| **자음·가성 보호** | 0.33 | 낮을수록 자음 보호↑. 0.50 = 비활성(기계음 원인!) |
| **볼륨 균일화(RMS)** | 0.15 | 0에 가까울수록 원곡 다이나믹 보존. 0.3↑ = 기계음 위험 |
| **음정 부드럽게(Filter)** | 3 | 높을수록 삑사리↓ 비브라토↓. 빠른 고음 곡은 2~3 |
| **피치 해상도(Hop)** | 128 | 대부분 128. 팔세토/고음 많으면 64 |
| **보컬 사전 피치 조정** | 0 | 여성 파트 포함 듀엣만 -5. 솔로 남성곡은 반드시 0 |
| **공간감(리버브)** | OFF | AI 건조함 제거용. 강도 0.05~0.10 권장 |
| **배음 강화** | OFF | 금속성 질감 완화. ON 권장 |
| **고음/가성 최적화** | OFF | 가성·팔세토 많은 곡에서 ON |
| **화음 구간 원본 유지** | **ON** | 화음/코러스 있는 모든 곡. 끄면 괴성 발생 |
| **팔세토·고음 원본 유지** | **ON** | 불안정 고음 구간 원본 대체. 기본 ON 유지 |
| **노이즈 구간 자동 원본유지** | **ON** | spectral flatness 기반 기계음 구간 자동 교체 |
| **여성 보컬 구간 원본 유지** | OFF | 남녀 듀엣곡에서만 ON (여성 파트 가래 방지) |
| **보컬 볼륨** | 1.0 | 목소리가 반주에 묻히면 1.2~1.5 |
| **반주(MR) 볼륨** | 1.0 | 반주가 너무 크면 0.8 |

### 곡 2: 플레이브 - 기다릴게

**파일명**: `플레이브 - 기다릴게.mp3`
**장르**: 한국어 K-POP, 남성 그룹, 고음·화음 있음, 44.1kHz MP3

#### 추천 프리셋: 1단계 🇰🇷 한국어 노래 → 고음 삑사리 시 🔥 고음곡 전용

**1단계 — 먼저 이 설정으로 테스트**:

| 설정 | 값 | 이유 |
|------|---|------|
| F0 방식 | **FCPE** | 고음/팔세토 안정성 최고. Docker 업데이트 후 사용 |
| 언어 | **한국어(ko)** | 비음(ㄴ/ㅁ/ㅇ) 포먼트 보호 |
| 인덱스 비율 | **0.75** | 권장 |
| 자음 보호 | **0.33** | 한국어 자음(ㅅ/ㅊ/ㅎ) 선명도 |
| RMS | **0.15** | 표준 |
| Filter | **3** | 표준 |
| Hop | **128** | 표준 (64로 낮추면 팔세토 더 정밀) |
| 피치 | **0** | 먼저 0으로 시작 |
| 화음 원본 유지 | **ON** | 그룹 화음 필수 |
| 팔세토 원본 유지 | **ON** | 고음 필수 |
| 노이즈 원본 유지 | **ON** | 기본 |
| 여성 보컬 원본 유지 | **OFF** | 남성 그룹 |
| 고음/가성 최적화 | **ON** | 고음 있는 K-POP |

**2단계 — 고음 삑사리/기계음 시**:
- 프리셋: 🔥 고음곡 전용 (pitch 자동 **-3** 설정됨)
- 또는 피치만 **-2**로 조정해 재변환

> **주의**: 고음곡 전용 프리셋은 pitch=-3으로 시작함. 원곡보다 낮아지면 -1~-2로 조정.

## 4. 변환 순서 (공통)

```
1. 변환 탭 클릭
2. 변환할 파일 선택 (업로드 탭에서 미리 업로드)
3. 모델 선택: My Voice v50 (또는 최신 학습 모델)
4. 프리셋 버튼 클릭 → 위 표 기준 세부 조정
5. "변환 시작" 클릭 → 완료 대기 (~2~4분/곡)
6. 결과 청취 → 아쉬운 부분만 조정 후 재변환
```

### 품질 문제별 빠른 조치

| 증상 | 조치 |
|------|------|
| 전체적으로 AI스러움 | 인덱스 비율 낮춤 (0.75 → 0.65 → 0.55) |
| 특정 구간 기계음 | 화음/팔세토/노이즈 원본 유지 ON 확인 |
| 고음 삑사리 | 피치 -2~-3, F0: FCPE, Hop: 64 |
| 자음 뭉개짐(ㅅ/ㅊ) | 자음 보호 낮춤 (0.33 → 0.25) |
| 여성 파트 가래 | 여성 보컬 원본 유지 ON, 보컬 사전 피치 -5 |
| 목소리가 반주에 묻힘 | 보컬 볼륨 1.2~1.5 |
| FCPE가 MR만 나옴 | Docker 이미지 최신화 필요 (v73 이후 빌드) |
